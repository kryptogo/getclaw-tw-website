你可能已經在付 ChatGPT Plus、Claude Pro 或其他 AI 服務的月費。這些訂閱通常包含一定的使用額度。**OAuth 策略的核心思路：讓 OpenClaw 直接使用這些訂閱額度，而不是另外付 API 費用。**

這不是破解，也不是灰色地帶。這是透過官方 OAuth 整合，讓你的訂閱價值最大化。

<Callout type="error">
**2026 Q2 重大景觀變化：Anthropic 已限制第三方工具使用 Claude 訂閱 Token。** 過去你可以執行 `claude setup-token` 把 Pro/Max 訂閱 Token 灌進 OpenClaw — 這條路目前對許多第三方工具已被切斷。**Claude Code 官方 CLI 的 OAuth 仍正常運作**，但 OpenClaw 這類第三方 Agent 平台需要評估替代方案：（1）改用 Anthropic API Key 按量計費；（2）切換到其他 provider（GPT-5.4 / Gemini 2.5 Pro / DeepSeek）；（3）混合策略 — 主力工作走 Claude Code OAuth、Agent 路由到其他模型。本頁以下內容假設你正在評估替代路徑。
</Callout>

<Callout type="warn">
**真實案例：$800 美元的教訓。** 即使 OAuth 路徑受限，**直接用 Anthropic Console 按量計費仍是最容易燒錢的路**。一位使用者分享他花了 $800 嘗試 Kimi、Opus、同時跑 8 個 Agent — 全部是按量計費。在做出 API Key 決定之前，先讀完[平價模型策略](/docs/cost/budget-models) — 模型路由（複雜任務走 Opus、日常走 Sonnet/Gemini Flash）能把月費砍半。
</Callout>

## Antigravity OAuth

### 它是什麼

Antigravity 是一個 OAuth 整合層，讓 OpenClaw 透過你現有的 AI 服務訂閱帳號來運作。你不需要另外申請 API Key，也不需要綁定信用卡到 API 計費帳戶。OpenClaw 會透過 OAuth 協議，以你的訂閱身份向 AI 服務發送請求。

### 運作原理

概念上很簡單：

1. 你在 OpenClaw 中設定 Antigravity OAuth 連線
2. OpenClaw 取得你的訂閱帳號授權
3. 之後的 AI 請求會透過你的訂閱額度處理，而非 API 計費

就像你用 Spotify 家庭方案 — 付一份月費，多個裝置都能聽。只不過這裡的「裝置」是 OpenClaw。

### 成本效益

以 ChatGPT Plus 為例：

- **訂閱費**：每月 ~$20（約 NT$650）
- **包含額度**：GPT-4o 使用量（有速率限制但日常夠用）
- **相比 API 直接計費**：同樣的使用量透過 API 可能要 $30-50+/月

你本來就在付這筆訂閱費。透過 Antigravity OAuth，這筆錢不再只是「用瀏覽器聊天」的費用，而是成為你整個 AI Agent 基礎設施的算力來源。

### 限制

不是萬能的：

- **速率限制**：訂閱方案有每小時/每天的請求上限，高頻使用會撞到天花板
- **模型可用性**：只能用該訂閱方案包含的模型
- **穩定性**：服務商可能調整訂閱條款或限制第三方存取

### 最適合誰

已經在付某個 AI 服務月費、使用量中等、想要多模型切換能力的個人使用者。

## Claude Code OAuth

### 它是什麼

跟 Antigravity 類似的概念，但專門針對 Anthropic 的 Claude 訂閱方案。如果你是 Claude Pro 或 Max 的訂閱者，可以透過這個整合讓 OpenClaw 使用你的 Claude 額度。

### 方案對比

Anthropic 目前提供幾個訂閱層級：

| 方案 | 月費 | Claude Sonnet 額度 | Claude Opus 額度 |
|------|------|-------------------|------------------|
| Pro | ~$20 | 標準額度 | 有限 |
| Max | ~$100 | 大量額度 | 較多 |

### 什麼時候值得用 Max

算一筆帳：如果你每月透過 API 使用 Claude Sonnet 的費用超過 $80，那 Max 方案的 $100/月反而更划算 — 因為你還額外得到 Opus 的使用額度。

但如果你的 Claude 使用量每月只有 $20-30 的 API 費用等級，那 Pro 方案就夠了，甚至直接用 API 計費可能更靈活。

### 與 Antigravity 的差異

- **專注度**：Claude Code OAuth 只能用 Anthropic 的模型，Antigravity 支援多家
- **整合深度**：因為專門針對 Claude 設計，整合體驗通常更順暢
- **額度計算**：Anthropic 的訂閱額度計算方式和其他服務不同，需要注意

## 策略比較

| | Antigravity OAuth | Claude Code OAuth | 直接 API 計費 |
|---|---|---|---|
| 每月成本 | 訂閱月費 | 訂閱月費 | 按 token 計費（變動） |
| 速率限制 | 依訂閱方案 | 依訂閱方案 | 付多少用多少（無硬上限） |
| 模型多樣性 | 多家供應商 | 僅 Anthropic | 任意供應商 |
| 成本可預測性 | 高（固定月費） | 高（固定月費） | 低（隨使用量波動） |
| 最適合 | 多模型使用者 | Claude 重度使用者 | 高頻/可預測使用量 |

## 怎麼選

**剛開始用 OpenClaw？** 如果你已經有任何 AI 服務訂閱，先用 OAuth 策略。零額外成本就能開始跑。

**使用量穩定成長？** 觀察你的使用模式。如果你發現自己經常撞到訂閱的速率限制，可能是時候考慮加入直接 API 計費作為補充。

**重度使用者？** OAuth 策略當作基礎額度，搭配平價模型的 API 計費處理溢出的請求。這是成本效益最高的混合策略。

## GetClaw 觀點

OAuth 策略過去是入門的最佳起點，但 2026 Q2 之後景觀重組了：**Anthropic 對第三方工具的訂閱存取已大幅收緊**。如果你還在規劃部署，務必把這個變化納入考量 — 不要照半年前的攻略走，那條路可能已經不通。

我們現在的建議分流：

- **重度 Claude 使用者** → 訂閱 Claude Pro/Max 走官方 Claude Code，OpenClaw 這邊掛 Anthropic API Key 處理 Agent 任務（兩條路線並行，互不取代）
- **預算敏感者** → 主力切到 Gemini 2.5 Pro（免費額度寬鬆）或 DeepSeek（極具競爭力的價格），保留少量 Anthropic API 給關鍵推理任務
- **企業部署** → 多 provider 混合 + 模型路由（複雜走 Opus、日常走 Sonnet/Flash），這是目前實測下成本最低、也最不被單一供應商綁架的架構
