## 三條路

### 1. 自己動手

你現在手上有完整的知識和工具。從[硬體選擇](/docs/hosting/options)開始，按照每個章節的步驟操作，遇到問題查[常見問題](/docs/faq)。

OpenClaw 社群很活躍，大部分問題在 24 小時內會有人回應。你不是一個人在摸索。

**適合你如果**：你享受動手的過程、有時間投入設定和維護、想完全理解系統的每一個環節。

### 2. 找 GetClaw

我們處理所有你不想操心的事：硬體選型與採購建議、完整部署、LINE 和 Telegram 頻道設定、安全強化、監控告警、持續維護、版本升級、問題排除。

你只需要專注在一件事上：**用 Agent 創造價值。**

我們見過太多團隊花了兩週把系統架起來，然後在第三個月因為維運負擔而放棄。不是因為技術不夠，是因為維運是一場持久戰，而你的時間應該花在更重要的地方。

**適合你如果**：你要的是結果而不是過程、時間成本高於金錢成本、或者你的組織需要 SLA 等級的可靠性。

### 3. 先聊聊

不確定哪條路適合你？這很正常。

預約一次免費諮詢，我們會根據你的使用場景、技術背景、預算和時間，給你一個具體的建議。不推銷、不施壓、不綁約。

有些人聊完之後決定自己來 — 我們會告訴他重點注意什麼。有些人聊完之後決定交給我們 — 我們會在一週內讓他的 Agent 上線。

**適合你如果**：你還在評估、想聽聽專業意見、或者有特殊需求想先確認可行性。

## GetClaw 觀點

自架 AI 是未來。這不是預測，是正在發生的事。

當你把 AI Agent 架在自己的機器上，你拿回的不只是資料的掌控權 — 你拿回的是決策權。用什麼模型、怎麼配 prompt、誰能存取、什麼時候升級 — 這些決策應該在你手上，不是在某個 SaaS 供應商的 roadmap 上。

我們寫這份指南，是因為相信一件事：**理解全貌的人，做出的決定更好。** 不管你最後自己動手還是讓我們來，你已經不再是那個只能被動接受「月費 $20、功能不夠就升級」的使用者了。

起步很簡單，做好需要經驗。

而我們每天都在累積這些經驗。
