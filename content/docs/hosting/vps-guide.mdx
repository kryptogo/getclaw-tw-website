## 先想清楚你要拿它做什麼

VPS 的規格不是「越高越好」，而是「剛好就好」。過度配置是浪費錢，配置不足會讓 Agent 卡頓。從你的實際使用場景出發，答案就很清楚。

### 場景一：純文字對話 Agent

你的 AI Agent 只需要接收訊息、呼叫 LLM API、回傳文字。LINE 和 Telegram 雙頻道同時運作，每天處理幾百則對話。

**你需要**：2 vCPU / 2GB RAM / 40GB SSD — 這是我們推薦的起步規格。OpenClaw Gateway 跑在 port 18789，Node.js >= 22 環境加上 JSON5 設定檔，2GB RAM 綽綽有餘，還有空間給日誌和暫存檔案。每月約 NT 150-300。

### 場景二：需要瀏覽器自動化

你的 Agent 要自動瀏覽網頁、擷取資料、填寫表單。Playwright + Chromium 會額外吃掉約 500MB RAM。

**你需要**：2 vCPU / 4GB RAM / 60GB SSD。多出來的記憶體不是奢侈，是必要。Chromium 在無頭模式下依然是記憶體大戶。每月約 NT 300-500。

### 場景三：只想先試試看

你想花最少的錢驗證 OpenClaw 能不能解決你的問題。

**最低能跑的規格**：1 vCPU / 1GB RAM / 20GB SSD。能跑，但幾乎沒有餘裕 — 記憶體緊張時 Node.js 的 garbage collection 會頻繁觸發，回應速度會受影響。拿來測試可以，正式環境不建議。每月低於 NT 150。

<Callout type="warn">
不管哪個場景，都確保你的 VPS 有獨立的 IPv4 位址。LINE Webhook 和 Cloudflare Tunnel 都需要 IPv4 連線能力。純 IPv6 的方案目前不適用。
</Callout>

## 供應商怎麼選

市面上 VPS 供應商很多，但對台灣用戶跑 OpenClaw 來說，真正重要的是三件事：機房離台灣多近、每月花多少錢、以及踩雷時有沒有人可以問。

| 供應商 | 最低月費 | 機房位置 | 升級彈性 | GetClaw 觀點 |
|---|---|---|---|---|
| Hostinger | ~USD 5 | 新加坡 | 中等 | 介面最友善，新手首選。台灣延遲低，社群使用量最大 |
| Hetzner | ~EUR 4 | 德國/芬蘭 | 高 | 歐洲 CP 值之王。延遲高（200ms+），但價格無人能敵 |
| DigitalOcean | ~USD 6 | 新加坡 | 高 | 開發者最愛。文件完整、API 好用、熱升級做得最好 |
| Linode (Akamai) | ~USD 5 | 東京 | 高 | 到 LINE 伺服器延遲最低（小於 10ms）。大品牌穩定度佳 |
| AWS Lightsail | ~USD 5 | 東京 | 極高 | AWS 生態系整合最強。未來想擴展到其他 AWS 服務很方便 |

### 機房位置怎麼想

LINE 的伺服器主要在日本，Telegram 分布全球。對台灣用戶而言：

- **東京** — 到 LINE 延遲小於 10ms，到台灣約 40-60ms。LINE 重度用戶的首選。
- **新加坡** — 到台灣 30-50ms，到 LINE（日本）70-90ms。LINE + Telegram 雙頻道的最均衡選擇。
- **歐洲** — 延遲 200ms 以上，但價格最低。如果你對延遲不敏感（例如非即時對話場景），可以考慮。

## 錢花在哪裡才值得

VPS 自架的月費經常被拿來跟 SaaS AI 工具比較，但這個比較其實低估了自架的價值。因為你買到的不是「一個有用量限制的聊天視窗」，而是一台完全可控的 AI Agent 主機。

| 你付的錢 | 你得到的東西 |
|---|---|
| ChatGPT Plus ~NT 650/月 | 一個聊天介面，有用量上限，不能自動化 |
| Claude Pro ~NT 650/月 | 同上，換了個品牌 |
| Notion AI ~NT 320/月 | 綁定 Notion 生態系的 AI 功能 |
| **VPS + OpenClaw ~NT 150-300/月** | **完整 root 權限的 AI Agent，24/7 自動運作，串接任意通訊平台，無用量限制** |

差別不只是價格 — 是架構層級的差異。SaaS 工具是別人的平台、別人的規則。自架是你的機器、你的 Agent、你的資料。

### 隱藏成本要算進去

VPS 月租費只是帳面成本。完整的 TCO（Total Cost of Ownership）還包括：

- **備份**：多數供應商提供自動備份，通常加收月租 20%。你的 Agent 的記憶和知識庫都在這台機器上，值得開啟。
- **流量**：OpenClaw 的流量消耗極低（主要是 JSON API 往返），多數方案的 1TB 月流量遠超所需。瀏覽器自動化會多一些，但通常也不會超標。
- **時間成本**：初始設定和偶爾的維護。這是最容易被忽略的成本 — 也是 GetClaw 服務最核心的價值。

## 作業系統與升級路徑

**作業系統選 Ubuntu 22.04 LTS 或 24.04 LTS。** LTS 版本有五年安全更新，社群資源最豐富，OpenClaw 的相容性測試也以 Ubuntu LTS 為主。

**升級路徑很重要。** 選支援熱升級的供應商 — 使用量成長時，不停機就能升級 CPU 和 RAM。DigitalOcean 和 Linode 在這方面做得最成熟。從 2GB 方案起步，需要時升到 4GB，整個過程不中斷服務。
