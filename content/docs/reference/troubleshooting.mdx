## Gateway 啟動失敗：Config validation failed

**原因：** 設定檔格式錯誤

```bash
# 查看錯誤訊息
sudo journalctl -u openclaw -n 30 --no-pager

# 檢查設定檔
cat ~/.openclaw/openclaw.json

# 如果 gateway.bind 有問題，改回 loopback
sed -i 's/"bind": "all"/"bind": "loopback"/' ~/.openclaw/openclaw.json
sudo systemctl restart openclaw
```

## LINE Webhook 返回 502 Bad Gateway

**原因：** Gateway 沒有在運行

```bash
# 確認 Gateway 狀態
sudo systemctl status openclaw

# 測試本地連線
curl -I http://127.0.0.1:18789/line/webhook
# 應該返回 405 Method Not Allowed（這是正常的，表示 Gateway 有在運行）
```

## DNS 無法解析：Could not resolve host

**原因：** DNS 還沒生效或記錄錯誤

```bash
# 用 Cloudflare DNS 測試
nslookup bot.<YOUR_DOMAIN> 1.1.1.1

# 如果有解析到 IP，但本機還是無法連，強制指定 IP
curl -I --resolve bot.<YOUR_DOMAIN>:443:<IP> https://bot.<YOUR_DOMAIN>/
```

## LINE Plugin 重複安裝警告

**訊息：** `duplicate plugin id detected`

**影響：** 通常不影響功能，可以忽略。

## Cloudflare Tunnel 路由到錯誤的網域

**原因：** `cloudflared tunnel login` 時選錯了網域

```bash
# 刪除舊憑證
rm ~/.cloudflared/cert.pem

# 重新登入，選擇正確的網域
cloudflared tunnel login

# 重新設定 DNS 路由
cloudflared tunnel route dns openclaw-tunnel bot.<YOUR_DOMAIN>
```

## VPS 上 curl 無法解析網域，但 nslookup 可以

**原因：** VPS 的 DNS 快取還沒更新

```bash
# 強制指定 IP 測試
curl -I --resolve bot.<YOUR_DOMAIN>:443:<CLOUDFLARE_IP> https://bot.<YOUR_DOMAIN>/

# 或等幾分鐘讓 DNS 快取自動更新
```

## Cloudflare Access 收不到驗證碼信件

**可能原因：**
1. 信件在垃圾郵件匣
2. Policy 沒有正確設定

**解決：**
1. 檢查垃圾郵件匣
2. 確認 Cloudflare Zero Trust → Applications → 你的 App → Policies 有設定 Allow + Emails

## Antigravity 版本不支援

**訊息：** `Antigravity version 1.11.5 is no longer supported`

```bash
# 找到檔案
find /usr -name "google-gemini-cli.js" 2>/dev/null

# 編輯檔案
sudo nano /usr/lib/node_modules/openclaw/node_modules/@mariozechner/pi-ai/dist/providers/google-gemini-cli.js

# 搜尋 User-Agent，把版本號改成最新版（如 1.15.8）
# 儲存後重啟
sudo systemctl restart openclaw
```
